## **RECOVERLUTION DESIGN SPEC v1**  
**Product UI as an operating system.** Not screens. **States. Loops. Receipts.**  
## **The purpose**  
Make the next right move obvious. Make return easy. Make proof inevitable. Make escalation calm.  
  
## **1) State Model (the real UI framework)**  
## **The four states (semantic)**  
* **CALM** — baseline building  
* **HEAT** — stress / urge / drift rising  
* **FRACTURE** — rupture / high risk / disorientation  
* **REPAIR** — return / reintegration / proof sealing  
## **The prime directive**  
**State first. Then story.** If state is high, reduce choice. Reduce language. Increase guidance.  
## **UI behavior by state (high-level)**  
**CALM:** more options, more reflection, longer copy allowed **HEAT:** fewer options, one CTA, short copy, fast path to a move **FRACTURE:** remove friction, escalate, safety rails, human-forward **REPAIR:** seal proof, reinforce without praise, set next gentle step

Every screen must show at least one of these:  
* **What’s happening** (READ)  
* **What’s next** (ROUTE/DELIVER)  
* **What just happened** (SEAL)  
If none of those are present, it’s not part of the OS. It’s decoration.

* **FRACTURE**: primary becomes human-forward (Contact / Call)  
* **REPAIR**: primary seals proof (Keep / Save / Replay)  
  
## **4.2 Cards**  
Cards are your “safety containers”.  
**Anatomy**  
* Truth line (title)  
* One sentence (human truth)  
* One action  
**State variants**  
* **CALM**: richer content, optional subtext  
* **HEAT**: only essential copy + single CTA  
* **FRACTURE**: higher contrast, fewer words, human CTA  
* **REPAIR**: becomes a receipt container  
  
## **4.3 NaviCue (Cue component)**  
This is the signature “moment meets system”.  
**Cue template**  
**Title:** A quick return? **Body:** 30 seconds. One move. **CTA:** Make a return  
**State variants**  
* **CALM**: cue can be optional suggestion  
* **HEAT**: cue becomes primary route (one tap into move)  
* **FRACTURE**: cue becomes “Bring in a human” gate  
* **REPAIR**: cue becomes “Seal the receipt”  
**Anti-patterns (never)**  
* guilt tone  
* streak references  
* “don’t forget”  
* urgency theatre  
  
## **4.4 Move (micro-intervention step)**  
Moves must feel: **doable · embodied · short · clean**  
**Move anatomy**  
* Step title (verb)  
* 1 instruction line  
* Timer/length indicator (optional)  
* “Why this, why now” (one line)  
* Completion action → creates receipt  
**Length rules**  
* **HEAT** move = 15–90 seconds  
* **CALM** move = 2–8 minutes  
* **FRACTURE** move = immediate grounding + escalation option  
* **REPAIR** move = 30–120 seconds + receipt  
**Move copy examples (style)**  
* **Breathe low.** 30 seconds.  
* **Name the urge.** One sentence.  
* **Change the room.** One step.  
* **Send one truth.** To one person.  
(Short. Human. No therapy jargon.)  
  
## **4.5 “Why this, why now” (Explainability)**  
Always one line. Always calm. Always non-technical.  
Templates:  
* “You’re in heat. Short moves land best here.”  
* “This is a return moment. Let’s keep it small.”  
* “You’ve done this before. We’re reinforcing it.”  
* “This pattern shows up in the evening. We’re routing earlier.”  
Never:  
* model talk  
* diagnostics  
* labels like “relapse risk high” in the UI  
  
## **4.6 Receipt (Proof object)**  
Receipts are sacred.  
**Receipt anatomy**  
* Title: Receipt of Return / Repair / Integrity  
* Fact line: “Returned in 48 seconds.”  
* Optional: “Held under pressure.”  
* Actions: Replay · Keep  
* Timestamp always visible  
**Receipt variants**  
* Return receipt  
* Repair receipt  
* Transfer receipt (life-held)  
* Trajectory receipt (baseline shift)  
**State variants**  
* **REPAIR**: receipt auto-prompts after completion  
* **CALM**: receipts appear as quiet confirmations  
* **HEAT**: receipts are minimal, no celebration  
* **FRACTURE**: receipts are replaced by safety confirmation + next step  
  
## **4.7 Proof strip (Receipt → Transfer → Trajectory)**  
Always small. Always visible. Never a dashboard.  
**Display rules**  
* show as 3 nodes  
* highlight current node  
* one micro line under it: “Default forming.”  
State variants:  
* **CALM**: show all 3 nodes, allow tap to learn  
* **HEAT**: show only current node + next  
* **FRACTURE**: hide; prioritize safety  
* **REPAIR**: animate a subtle “node fill” to reinforce completion  
  
## **4.8 Check-in (State capture)**  
Check-ins must not feel like forms.  
**Check-in rules**  
* 1–3 inputs max  
* no long questionnaires  
* no “rate your day” vibes  
* always ends with a routed move or a “no action needed” calm confirmation  
Copy examples:  
* “Where are you right now?”  
* “Do you want a small return?”  
* “Keep it quiet today?”  
  
## **4.9 Escalation card (Human handoff)**  
This is how you do safety without fear.  
**Copy template**  
**Bring in a human.** This moment deserves care. CTA: Contact support  
Rules:  
* no alarm graphics  
* no red screaming  
* serious ≠ scary  
  
## **5) Layout Rules (stress-proof design)**  
## **CALM layout**  
* two actions allowed  
* optional exploration  
* longer copy blocks  
## **HEAT layout**  
* one column  
* one CTA

* no menus  
* no browsing

## **FRACTURE layout**

* seal proof immediately

## **6) UI Copy System (microcopy bible)**  
## **Tone constraints**  
* calm, factual, kind  
* never praise-heavy  
* never shame-coded  
* never “you should”  
## **Preferred phrasing**  
* “Let’s make a small return.”  
* “Keep it simple.”  
* “One move.”  
* “This is enough.”  
* “We’re reinforcing what worked.”  
## **Disallowed phrasing**  
* “Great job!” (too performative)  
* “Don’t break your streak.”  
* “You failed.”  
* “Noncompliant.”  
* “Relapse risk high.” (in user UI)  
  
## **7) Accessibility Standards (premium = accessible)**  
## **Contrast**  
* Text must pass WCAG AA minimum.  
* In HEAT and FRACTURE, raise contrast further (stress reduces perception).  
## **Typography**  
* Minimum body size: 16px  
* Line height: 1.5–1.7  
* Avoid all caps in body (caps only for tiny labels if needed)  
## **Motion**  
* Provide prefers-reduced-motion fallbacks  
* No essential info conveyed only by motion  
* No flashing, no strobing  
## **Touch**  
* 44px minimum tap targets  
* Spacing around CTAs in HEAT/FRACTURE increased  
## **Cognitive load**  
* One decision at a time  
* Progressive disclosure  
* Never bury help  
  
## **8) Console + Command Center (pro rules)**  
## **Professionals (Console)**  
* “Signal, not noise” is the principle  
* Default view: **what held / what slipped / return speed**  
* No surveillance tone  
* Proof objects are clinically legible, not performative  
## **Organisations (Command Center)**  
* Outcomes shown as **defensibility**

* Audit-ready posture without “dashboard porn”  
* Human capacity preserved as a design KPI

## **Night-Lux (default)**

* accent is minimal

* best for CALM, reading, professional/org contexts

* lots of white space  
**Rule:** theme must not change meaning. Only ambience.
